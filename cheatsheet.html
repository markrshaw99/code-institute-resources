HTML & CSS Cheat Sheet

Day 1: Introduction to HTML + Semantic HTML

HTML (HyperText Markup Language) is the backbone of web content. It defines the structure of your webpage. Semantic HTML uses meaningful tags to help both browsers and assistive devices understand your content better. For example, using <header> for headers or <main> to denote the primary content improves accessibility and SEO.

HTML Document Structure


<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Website</title>
  </head>
  <body>
    <h1>Main Heading</h1>
    <p>A paragraph.</p>
  </body>
</html>

Semantic Elements

Using semantic tags like <header>, <nav>, <main>, <article>, <section>, <aside>, and <footer> adds clarity and meaning to your markup.


<header>Website Header</header>
<nav>Main Navigation</nav>
<main>Main Content</main>
<article>Blog Post</article>
<section>Logical Section</section>
<aside>Sidebar Content</aside>
<footer>Page Footer</footer>

Analogy: Think of semantic tags like well-labelled folders in a filing cabinet – everything is in its right place for screen readers, search engines, and developers.

Day 2: Lists and Links

Lists are useful for grouping related items together, such as navigation menus or features. Links (<a>) connect your page to additional content or external resources. The rel="noopener" attribute increases security when opening links in new tabs.

Lists


<ul>
  <li>Unordered Item 1</li>
  <li>Unordered Item 2</li>
</ul>

<ol>
  <li>Ordered Item 1</li>
  <li>Ordered Item 2</li>
</ol>

Links


<a href="https://example.com" target="_blank" rel="noopener">Visit Site</a>

Day 3: Images and Attributes

Images add visual flair to your website. Always include the alt attribute for accessibility and use loading="lazy" to defer loading until the image is visible in the viewport.


<img src="cat.jpg" alt="A sleeping cat" width="300" loading="lazy">

Day 4: CSS Introduction + Selectors

CSS (Cascading Style Sheets) is used to style your HTML content. It can be added inline, internally, or externally. Selectors target specific elements of your webpage so you can control their appearance.

CSS Selectors


/* Type Selector */
p { 
  color: blue; 
}

/* Class Selector */
.button { 
  background: black; 
}

/* ID Selector */
#header { 
  text-align: center; 
}

/* Descendant Selector */
nav a { 
  text-decoration: none; 
}

/* Pseudo-class */
a:hover { 
  color: red; 
}

Day 5: Colours, Units, and Typography

Colours, measurement units, and typography all contribute to the overall design of your site.

Colours

  • Named: e.g., red, blue
  • Hex: e.g., #ff6600
  • RGB: e.g., rgb(255, 0, 0)
  • HSL: e.g., hsl(0, 100%, 50%)

Units

  • Absolute: e.g., px, cm
  • Relative: e.g., em, rem, %, vw, vh

Typography


body {
  font-family: 'Roboto', sans-serif;
  font-size: 1rem;
  line-height: 1.6;
}

To include fonts, you might import one from Google Fonts:


<link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet">

Day 6: Box Model

The box model is a key concept in CSS. Every element is a box composed of content, padding, border, and margin. Mastering the box model is crucial for precise layouts.


div {
  margin: 10px;
  border: 5px solid #000;
  padding: 20px;
  width: 200px;
  box-sizing: border-box;
}

Analogy: Think of it like a package—the content is your item, the padding is the protective bubble wrap, the border is the box, and the margin is the space outside the package.

Day 7: Layout – Display and Positioning

Layout properties like display and position determine how elements are arranged on your page. They give you control over alignment, spacing, and stacking.

  • Display: e.g., block, inline, inline-block, none
  • Position: e.g., static, relative, absolute, fixed, sticky
  • Float & Clear: Used for wrapping text and creating multi-column layouts.
  • Visibility & Overflow: Control what happens when content exceeds its container.

Day 8: Forms + Basic Validation

HTML forms capture user input. HTML5 validation attributes like required, minlength, maxlength, and pattern help perform basic checks without JavaScript.


<form action="/submit" method="post">
  <label for="email">Email:</label>
  <input id="email" type="email" name="email" required>
  <input type="submit" value="Send">
</form>
        

<input type="text" name="username" required 
       minlength="3" maxlength="15" pattern="[A-Za-z0-9]+">

Day 9: Flexbox

Flexbox simplifies creating flexible, responsive layouts. It controls spacing, alignment, and distribution of items in a container.


.container {
  display: flex;
  flex-direction: row;
  justify-content: space-between;
  align-items: center;
}
        

.item {
  flex: 1 1 200px; /* grow, shrink, basis */
}

Day 10: Media Queries + Responsive Design

Responsive design ensures your site performs wonderfully on every device. Begin with the viewport meta tag and then refine your layout with media queries.


<meta name="viewport" content="width=device-width, initial-scale=1.0">
        

@media (max-width: 768px) {
  .menu {
    flex-direction: column;
  }
}

Day 11: Tables + Multimedia

Tables help display structured data, while multimedia elements like video and audio enhance interactive content.


<table>
  <thead>
    <tr><th>Name</th><th>Age</th></tr>
  </thead>
  <tbody>
    <tr><td>Alice</td><td>30</td></tr>
  </tbody>
</table>
        

<video controls width="300">
  <source src="movie.mp4" type="video/mp4">
</video>

<audio controls>
  <source src="song.mp3" type="audio/mpeg">
</audio>

Bonus: CSS Grid, Transitions & Accessibility

This section delves into advanced features like CSS Grid for two-dimensional layouts, smooth property transitions and animations, and accessibility best practices.

CSS Grid


.grid-container {
  display: grid;
  grid-template-columns: 1fr 2fr;
  gap: 10px;
}

Transitions & Animations


button {
  background-color: blue;
  transition: background-color 0.3s ease;
}

button:hover {
  background-color: green;
}

Accessibility Basics


<nav aria-label="Main menu">
  ... navigation elements ...
</nav>
